Introduction

BrightQuery (BQ) can contribute to the Genesis Mission by providing large-scale, structured, and interoperable data assets (via its global entity graph and OpenData.org initiative) that directly support the mission’s emphasis on data integration, standards, and AI-ready datasets. BQ's expertise in entity resolution, data normalization, and real-time data pipelines enables the creation of high-quality, linked datasets essential for training and validating advanced AI models across scientific domains. In addition, BQ's Factual AI capabilities—combining knowledge graphs, retrieval-augmented generation (RAG), and continuous data validation—support the Genesis Mission’s goal of building an integrated AI discovery platform that connects datasets, models, and compute resources to accelerate research and innovation. This aligns with the consortium’s focus on AI model development, data interoperability, and high-performance computing ecosystems. At a strategic level, BQ can serve as a data infrastructure and standards layer within the consortium, helping ensure that disparate datasets across DOE labs, industry, and academia are machine-readable, linkable, and usable by AI systems—ultimately improving the speed, reliability, and reproducibility of scientific discovery.

Offerings & Experience

Offerings
AI models, tools, or software

BQ applies retrieval-augmented generation (RAG), graph-based reasoning (GraphRAG), and continuous data validation to improve model accuracy, explainability, and factual grounding—capabilities that are critical for scientific and mission-critical applications. BQ's pipelines support real-time data ingestion, normalization, and enrichment, ensuring that models operate on current and high-quality information.

AI/ML/Data science expertise

BQ brings a tightly integrated AI/ML and data science stack that directly supports the Genesis Mission’s objective of accelerating scientific discovery through interoperable data and advanced analytics. BQ specializes in large-scale data engineering and entity resolution, transforming fragmented public and private datasets into unified, AI-ready knowledge graphs. These structured datasets enable downstream machine learning workflows, including feature engineering, model training, and validation across heterogeneous data sources.

Datasets or data infrastructure

BQ curates and maintains large-scale, structured data assets derived from government filings, open data sources, and web-scale ingestion pipelines. These datasets are normalized, standardized, and continuously updated, enabling consistent use across domains. Through its OpenData.org initiative, BQ also promotes open, shareable data ecosystems, aligning with Genesis’s emphasis on collaborative, cross-institutional data access. At the infrastructure level, BQ builds a global entity graph that links organizations, people, and locations via persistent identifiers.

Relevant Projects

(1) https://www.americasdatahub.org/ai-rd-24/

This project explores how to make agencies’ statistical data products more readily ingestible by AI technologies. It will produce an AI readiness assessment as a shared resource for any agency looking to test the machine understandability of its public data products and an AI readiness prototype tool to transform public data products into machine-understandable, AI-ready data.

(2) https://www.americasdatahub.org/data-access-alternatives-artificial-intelligence-supported-interfaces-daa/

This project seeks to develop and pilot an AI “chatbot” that answers natural language user questions based on public data products from federal statistical agencies. In the first part of the pilot, the team is building a Retrieval Augmented Generation (RAG) based system that is compatible with and builds on the open-source framework behind Google’s Data Commons. The chatbot will focus on types of data products that represent how statistical agencies publish public data.(1) public use files, (2) data tables, and (3) analytical reports. These features are designed to make public data more accessible, useful, and relevant for a broad range of users, including those in science, policy, journalism, and more. In addition to a pilot tool, this project will record lessons learned about the size of input data tables, making statistical data “AI ready”, and engineering issues encountered while building the pilot tool.

(3) https://www.americasdatahub.org/rfs-dup-sfa-25/

The key objective of this project is to develop an easy-to-maintain, user-friendly dashboard showing the utilization of federal data by a wide variety of data users with varying experience in research, decision-making and policymaking. Federal data can include administrative data as well as statistical data. Currently, there is no shared resource for federal agencies to track the usage of their data. Related works include the Democratizing Data Initiative and the Federated Data Usage Platform project. This project seeks solutions to create and maintain a shared data usage tracking dashboard as part of the National Secure Data Service (NSDS) that will provide data users information on the timely and relevant usage of federal data.

(4) https://www.americasdatahub.org/pcs-dcs-25/

The objective of this project is to develop a data concierge service that provides high-quality technical assistance to help data users with data discovery, access, and linkage as well as navigating services available within a National Secure Data Service (NSDS).
