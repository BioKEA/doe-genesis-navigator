    Relevant Projects

    Project 1: Multi-LLM Orchestration for High-Quality Code Generation (PerfOrch) We investigated how complementary strengths across frontier LLMs can be exploited for code generation through intelligent orchestration rather than single-model pipelines. We designed PerfOrch, a four-agent system (categorization, generation, debugging, refinement) where each agent consults offline-profiled memory rankings indexed by programming language and algorithmic category to route tasks to the most suitable LLM. We evaluated on 2,500 problems across five languages (Python, Java, C++, Go, Rust) and two benchmarks. PerfOrch achieved 97.19% and 95.83% average pass@1 on HumanEval-X and EffiBench-X respectively, improving over the strongest single-model pipeline by up to 14.58 percentage points. Memory rankings built from one benchmark generalized to an entirely unseen benchmark without re-profiling. PerfOrch matched exhaustive multi-model refinement coverage at roughly half the token cost. Manuscript submitted to ACM TOSEM.

    Project 2: Self-Supervised Semantic Segmentation via Multi-Scale Structural Complexity (MSSSeg) We addressed structural errors in self-supervised semantic segmentation—merging distinct objects or fragmenting coherent regions—that arise from reliance on low-level appearance cues. We developed MSSSeg, introducing three coupled components: a Differentiable Box-Counting module extracting multi-scale structural complexity from semantic and depth domains, a Learnable Structural Augmentation corrupting pixel-intensity patterns to force reliance on structural features, and a Persistent Homology Loss supervising connectivity at critical pixels. MSSSeg achieved new state-of-the-art on COCO-Stuff-27 (34.9 mIoU, ViT-B/8), Cityscapes (26.8 mIoU), and Potsdam (75.7 mIoU) without excessive computational overhead. Targeting ECCV 2026.

    Project 3: Physics-Informed Monocular Depth Estimation in Challenging Environments (PhysDepth) We identified that state-of-the-art monocular depth estimation models fail under adverse conditions (nighttime, rain) because they learn brittle, non-physical heuristics rather than robust physical principles. We proposed PhysDepth, a plug-and-play framework integrating Rayleigh Scattering theory and Beer-Lambert law into existing MDE backbones. The framework includes a Physical Prior Module extracting features from the high-SNR red channel and a Red Channel Attenuation Loss enforcing physically consistent depth-attenuation relationships. PhysDepth achieved SOTA on RobotCar-Night (0.109 AbsRel), nuScenes-Night (0.118 AbsRel), and nuScenes-Rain (0.130 AbsRel), and demonstrated consistent improvements when plugged into six different existing backbones, confirming model-agnostic generality.

    Project Idea Summary

    We propose AI-driven perception for autonomous DOE experimental workflows under extreme lighting conditions in facilities. DOE facilities—synchrotron beamlines, hot cells, high-energy-density platforms—require repetitive, high-risk sample manipulation that remains manual. The bottleneck is perceptual: extreme dynamic range (>100 dB), radiation-induced sensor corruption, and microsecond transients defeat conventional computer vision, blocking safe robotic autonomy.

    Our approach embeds image-formation physics (optical flow, HDR response curves, radiation noise statistics) as differentiable constraints within neural fields, enabling physically consistent feature recovery. The system comprises: (1) physics-constrained feature restoration via content-motion neural fields with HPC-accelerated training on digital-twin synthetic data; (2) uncertainty-aware 3D scene understanding through dual-stream fusion producing HDR images, metric depth, and occupancy grids with calibrated uncertainty; and (3) a perception-to-action interface translating uncertainty-quantified 3D representations into confidence-annotated manipulation decisions with safety-triggered human oversight.

    We seek partners with DOE facility access (beamlines, hot cells, or high-energy-density platforms) who can provide deployment environments, domain-specific sensor data under extreme conditions, and robotic infrastructure for validation.
