    Introduction

    TaskHawk Systems builds formally verified AI governance infrastructure. Our core product, the Kevros Enforcement Kernel, is a permission-before-power architecture that mediates between autonomous AI agents and consequential action. The kernel has been verified across six independent tools including TLA+ (1.94 billion states), Lean 4 (20 theorems, zero unproven assumptions), and four additional verification layers, totaling 71 proofs with zero failures. The architecture applies to any domain where an autonomous system must be provably constrained before executing high-consequence operations. We hold a CAGE code, active SAM.gov registration, and CMMC Level 2 self-attested and recorded at 110/110. Published research and verification artifacts are publicly available.

    Relevant Projects

    Project 1: Six-Layer Formal Verification of an AI Governance Enforcement Kernel (2025-2026)

    Conducted the first published six-layer formal verification of a single AI governance product, establishing a methodology for verifying safety-critical enforcement boundaries in autonomous systems. The verification campaign confirmed 12 safety invariants and 4 liveness properties across 1.94 billion reachable states using TLA+/TLC exhaustive model checking, Amazon Kani/CBMC bounded model checking (17 harnesses), Microsoft Verus/Z3 unbounded deductive proof (22 proofs), runtime assertion checking, cross-language byte-identity equivalence testing, and Lean 4 interactive theorem proving (20 theorems, zero unproven assumptions). The combined result was 71 independent proofs with zero violations at a total compute cost under $4. Key contributions include a novel algebraic region abstraction that eliminates IEEE 754 floating-point arithmetic from bounded model checking, reducing SAT solving time from over 35 minutes to 33 seconds, and a bisimulation relation enabling shared abstract models across SAT and SMT verification backends. All specifications, proofs, and verification artifacts are publicly available under MIT license.

    Project 2: Commercial Deployment of Cryptographic AI Governance Infrastructure (2025-2026)

    Designed, implemented, and commercially deployed a cryptographic policy enforcement layer for autonomous AI agents, listed as transactable offerings on both Microsoft Azure Marketplace and AWS Marketplace. The system implements a permission-before-power architecture incorporating post-quantum digital signatures (ML-DSA-87 per FIPS 204), hash-chained provenance ledgers, and behavioral drift detection. A co-published Microsoft press release accompanied the Azure listing. Production endpoints supporting x402 (USDC stablecoin) and L402 (Lightning Network) payment protocols are indexed on the 402 Index public directory with 100% measured uptime over 30 days. The deployment maintains CMMC Level 2 compliance at 110/110 controls as self-attested and recorded. The organization holds NIST NCCoE DevSecOps Community of Interest membership (established July 2025), a CAGE code, and active SAM.gov registration, establishing federal procurement readiness.

    Project 3: Formal Specification for Multi-Protocol Autonomous Agent Payment Governance (2026)

    Authored the Agent Budget SDK Specification, a 1,436-line formal type system defining wire formats and runtime interfaces for multi-rail autonomous agent payment governance. The specification addresses budget delegation from operators to agents, payment execution across heterogeneous settlement networks (L402/Lightning, x402/USDC, MPP/Stripe-Tempo, ACP/Stripe), cryptographic sub-agent budget partitioning with monotonic attenuation, and hash-chained audit trail generation with governance kernel integration. A comprehensive market analysis identified 10 previously undocumented technical gaps in agent payment infrastructure, each subsequently addressed in the specification with normative type definitions. Additionally, designed and published an open-source multi-protocol crawler and discovery schema for the 402 Index, cataloging 16,258 paid API endpoints across three payment protocols with automated health scoring, schema validation, and cross-protocol discovery. The specification provides the foundation for a planned IETF Internet-Draft submission.
